## 📋 步骤 0：项目背景说明

```
我需要开发一个教育类前端 Demo，用于展示 1 对 1 在线英语单词课堂的教学流程。

这个系统有两个角色：
1. 学生端：看到学习内容，做题，输入答案
2. 教师端：监控学生状态，控制教学节奏，提供帮助

重要特征：
- 这是一个纯前端展示项目，用假数据模拟即可
- 学生端和教师端需要实时同步（比如学生选了 A 选项，教师端立刻看到）
- 有固定的页面布局：左边 70% 是内容区，右边 30% 是两个视频窗口
- 视频窗口是固定的，永远不能被遮挡或移动

请确认你理解了这个项目的基本概念。
```

## 📋 步骤 1：核心教学逻辑（⚠️ 最重要）

```
在开始写代码之前，我需要你理解这个课堂的教学逻辑。

### 1. 单词有三种状态（像红绿灯）

- 红灯词：学生还不会，需要重点攻坚
- 黄灯词：基本掌握，需要定期复习
- 绿灯词：完全掌握了

### 2. 每个单词有 6 个维度的学习内容

- 核心：单词本身、音节拆分（比如 a·dapt）
- 声音：音标、音频
- 意义：中文意思、英文解释、同义词
- 语境：3 个例句（简单、中等、复杂）
- 逻辑：记忆口诀、词根拆解（比如 ad- 表示 to，apt 表示 fit）
- 视觉：图片描述、记忆联想

### 3. 有两种上课模式

**模式 A - 标准新授课**（教新单词）：
- 第 1 阶段：精准筛查（快速测试学生认不认识）
- 第 2 阶段：AI 训练（三种练习：听音、闪视、拼写）
- 第 3 阶段：门神验收（全拼写考核）

**模式 B - 攻坚复习课**（专门复习红灯词）：
- 步骤 1：听音（播放音频，可选显示音标）
- 步骤 2：看形（显示音节拆分）
- 步骤 3：探因（显示词根和记忆口诀）
- 步骤 4：书写（让学生完整拼写）

### 4. 关键教学原则（必须遵守）

1. **批量推进原则**：
   - 假设有 5 个单词，是 5 个词一起完成第 1 阶段，再一起进入第 2 阶段
   - 不是 1 个词完成所有阶段，再做下一个词

2. **第 2 阶段的三轮训练顺序**：
   - 5 个词先全部完成"听音辨形"
   - 然后 5 个词全部完成"闪视辨析"
   - 最后 5 个词全部完成"幽灵拼写"

3. **第 3 阶段必须包含所有单词**：
   - 不管第 1 阶段答对还是答错，所有单词都要通过第 3 阶段的全拼写验收

4. **听音辨形不能显示单词**：
   - 这道题是测试学生能不能通过听音认出单词
   - 如果直接显示单词文本，就泄题了

5. **教师可以随时提供帮助**：
   - 教师端有一个"武器库"，固定在所有页面上
   - 教师随时可以点击"拆音节""读口诀""讲词根""看图片"
   - 学生端立刻弹出一个卡片，显示对应的帮助内容

请用你的话复述一遍这些教学逻辑，确保你理解正确。
```

## 📋 步骤 2：页面布局和导航

```
现在开始设计页面结构。

### 1. 页面布局（固定不变）

所有页面都用这个布局：
- 左边 70%：内容区（显示题目、输入框、控制按钮等）
- 右边 30%：双视频窗
  - 上面的视频：对方（学生端显示老师，教师端显示学生）
  - 下面的视频：自己

⚠️ 重要：视频窗是固定的，任何功能都不能遮挡或移动它们。

### 2. 需要的页面

1. **首页**：
   - 选择上课模式（模式 A 或 模式 B）
   - 选择查看视角（学生视角 / 教师视角 / 双屏对比）
   - 模式 A 显示 6 个阶段的流程图
   - 模式 B 显示 4 个步骤的流程图

2. **学生端页面**：
   - 顶部：显示当前在第几阶段（1-6 的进度条）
   - 内容区：根据当前阶段显示不同内容（题目、卡片、输入框）
   - 视频区：上方显示老师，下方显示自己

3. **教师端页面**：
   - 顶部：显示学生当前状态（在第几阶段、第几个单词）
   - 内容区：监控面板 + 控制按钮（下一题、重做、显示答案等）
   - 右上角：固定的"武器库"面板（4 个按钮）
   - 视频区：上方显示自己，下方显示学生

4. **双屏演示页面**：
   - 左边：学生端
   - 右边：教师端
   - 方便展示时对比两边的界面

### 3. 设计风格

- 品牌颜色：51Talk 的橙色（主色调）
- 状态颜色：红色（不会）、黄色（基本会）、绿色（掌握）
- 圆角：按钮用小圆角，卡片用中圆角
- 阴影：卡片有轻微阴影，弹窗有明显阴影

请开始创建项目结构，实现这个基础布局。
```

## 📋 步骤 5：实现模式 A - 第 2 阶段（三轮训练）


## 📋 步骤 6：实现模式 A - 第 3 阶段（门神验收）

```
第 3 阶段是最终验收，要求学生完整拼写单词。

### 重要规则

1. 所有 5 个单词都要进入第 3 阶段（不管第 1 阶段答对还是答错）
2. 每个单词有 2 次尝试机会
3. 通过验收 → 标记为"黄灯词"（基本掌握）
4. 2 次都失败 → 标记为"待复习"（将来要回炉第 2 阶段）

### 学生端界面

1. 显示一个例句，目标单词位置是空白
   - "We must _____ our strategy to the market."
   
2. 下方显示例句的中文翻译
   - "我们必须调整策略以适应市场。"

3. 一个输入框（空白，无任何提示）

4. 提交按钮

5. 答题逻辑：
   - 第 1 次答对 → 显示"通过验收"，自动跳到下一个单词
   - 第 1 次答错 → 显示"还有 1 次机会"，清空输入框
   - 第 2 次答对 → 显示"通过验收"，跳到下一个单词
   - 第 2 次答错 → 显示"需要继续复习"，跳到下一个单词

### 教师端界面

1. 显示"第 3 阶段 - 门神验收"
2. 显示当前单词和尝试次数
   - "adapt (第 1 次尝试)"
3. 实时显示学生输入
4. 显示验收结果统计
   - 通过：3 个
   - 待复习：2 个

### 全部完成后

5 个单词都验收完成后，显示：
- "模式 A 课程结束"
- 结果统计：通过 X 个，待复习 X 个

请实现这个第 3 阶段的功能。
```
